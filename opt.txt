";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";s:5:"child";a:2:{s:0:"";a:5:{s:5:"title";a:1:{i:0;a:5:{s:4:"data";s:43:"WPTavern: Review Of WP-ElasticSearch Plugin";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"guid";a:1:{i:0;a:5:{s:4:"data";s:31:"http://www.wptavern.com/?p=6980";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"link";a:1:{i:0;a:5:{s:4:"data";s:95:"http://feedproxy.google.com/~r/WordpressTavern/~3/1do2CNarE8M/review-of-wp-elasticsearch-plugin";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:11:"description";a:1:{i:0;a:5:{s:4:"data";s:5651:"<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/SearchBoxIOLogo.jpg" rel="thumbnail"><img class="alignright size-full wp-image-6981" alt="SearchBoxIO Logo" src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/SearchBoxIOLogo.jpg?resize=193%2C37" /></a>On my recent article <a href="http://www.wptavern.com/searching-for-a-better-wordpress-search">discussing the search functionality</a> within WordPress, a few folks in the comments suggested alternatives to try. One of those was the <a href="http://wordpress.org/extend/plugins/wp-elasticsearch/">ElasticSearch</a> plugin. My main complaint in that article was that there was not an easy way to tap into the Elastic Search service without having access to a dedicated server or VPS. This is where SearchBox.IO comes in handy. <a href="https://searchbox.io/">SearchBox.IO</a> is a cloud based search platform powered by the Elastic Search technology. While there is no mention of a WordPress plugin on their site, one does exist.</p>
<p>Installing the plugin from the repository was easy, once I found it. After installation, you&#8217;ll need to connect the plugin to the Searchbox API server. On the configuration screen, it&#8217;s not obvious as to where you would get the API key. However, the <a href="http://wordpress.org/extend/plugins/wp-elasticsearch/installation/">installation instructions</a> on the WordPress.org plugin page tell you where to go. Once you sign up for a free account, you&#8217;ll need to copy and paste the <strong>Connection URL:</strong> provided into the <strong>Elastic Server url</strong> box.</p>
<p><a href="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ElasticSearchConfig.jpg" rel="thumbnail"><img class="aligncenter size-full wp-image-6982" alt="Elastic Search Configuration" src="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ElasticSearchConfig.jpg?resize=500%2C258" /></a></p>
<p>Once a connection is established, you&#8217;ll see a green server status indicator. The next step is indexing posts. Before you click the Index button, you&#8217;ll need to make sure that the <strong>Default Post Index Name</strong> within the <strong>Indexing Configurations</strong> matches the index name on your Searchbox.IO account. By default, it&#8217;s <strong>WordPress</strong> but I changed mine to WPTavern. It&#8217;s worth noting that the free account only allows <em>one Index, 3,000 documents, 10 MB of storage and something called a sleepy index</em>. After the site is indexed, you&#8217;ll want to visit the widgets section and place the Elasticsearch Facet Widget at the top of the sidebar. It will only be displayed when someone searches your site.</p>
<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ElasticSearchWidget.jpg" rel="thumbnail"><img class="aligncenter size-full wp-image-6983" alt="ElasticSearchWidget" src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ElasticSearchWidget.jpg?resize=306%2C361" /></a></p>
<p>The widget adds the ability to narrow down a search via <strong>Tags</strong>, <strong>Categories</strong>, or <strong>Author</strong>. At this point, it&#8217;s difficult to tell when your site is using Searchbox.IO for the search query or if it&#8217;s using the local database. I have not found an easy way of figuring this out. I did send in a support query asking for some clarification, here was the response:</p>
<p><strong>Q: I&#8217;ve installed and configured the WP-Elastic search plugin. However, I am having a hard time figuring out when searches are offloaded to the Searchbox.io service versus when my local database is polled. Could you please clarify this for me?</strong></p>
<p><strong>A: Plugin simply searches at searchbox and executes get requests to mysql with returned ids. Hope it is clear, let me know if you have more questions.</strong></p>
<p>I have no idea what that means so if you do, please tell me in the comments.</p>
<h2>Conclusion:</h2>
<p>After everything was said and done, I couldn&#8217;t tell whether the searches I performed were any more relevant than the default search in WordPress. However, this plugin tied into Searchbox.IO does add the features I&#8217;m looking for in a more advanced WordPress search. One thing I really didn&#8217;t like is that each time a check box is selected for a tag, category, or author, it generated another search query. This doesn&#8217;t make any sense to me. I&#8217;d much rather see an implementation where I can configure those parameters then click on a search button or hit enter. This plugin along with the Searchbox.IO website could use more polish. It would be nice to see an entire page on their website dedicated to WordPress that explains what the plugin does, how to set it up and ultimately, how to know if it&#8217;s working.</p>

One of the issues deals with the attractiveness of the WordPress back-end.</p>
<blockquote><p>Additionally, the drawbacks of the WordPress platform do not stop at its growing complexity. It’s also notoriously ugly. Now don’t get me wrong, if you know me then you know I’m a huge WordPress fanboy who makes the majority of his income each month from writing solely on and about the WordPress platform. So I have crazy amounts of love, respect and appreciation for this software. But with that said I’ve often wondered: who stole all the sexy? I mean, seriously! Why does the backend of this amazing tool have to look SO dull and boring?!</p></blockquote>
<p>I agree with him to an extent. The WordPress back-end is grey, dull and boring although using the different color scheme of blue, things brighten up a bit. But then, the entire back-end becomes too blue. Other than the colored text in the Discussion area of the Right Now box, there is very little in the way of refreshment in the back-end of WordPress. What happened here?</p>
<p>If we rewind our <a href="http://wordpress.org/news/2008/10/the-visual-design-of-27/">calendars to October 2008</a>, WordPress was undergoing a major UI change from 2.5 codenamed CrazyHorse. This new UI was described as sexy, awesome, Wow! out of the box, etc. It was definitely a large improvement over WordPress 2.5 both in usability and looks. While I can&#8217;t find the article which contained the screenshots, there was at least one instance where Joomla showcased a screenshot of a new UI that was designed that looked very similar to the back-end of WordPress which was flattering to say the least.  </p>
<p><a href="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wordpress2.7Dashboard.jpg" rel="thumbnail"><img src="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wordpress2.7Dashboard.jpg?resize=500%2C333" alt="WordPress 2.7 Dashboard" class="aligncenter size-large wp-image-6975" /></a></p>
<p>The back-end has gone through design tweaks since then and with WordPress 3.7, <a href="http://www.wptavern.com/mp6-is-like-eye-candy">it appears</a> that more user interface design changes are in the pipeline. In my opinion, for the better. Looking at the current iteration of the back-end UI, what would make it more delightful to the eyes or refreshing to use so that it wasn&#8217;t so boring? </p>
<p>One of other sticking points is the bit about Ghost being a true non-profit.</p>
<blockquote><p>Ghost will be a true non-profit. There will be no commercial side to things as there currently is with Automattic and WordPress</p></blockquote>
<p>What does it mean to be a non-profit but then be classified as not a true non-profit. The WordPress project is not aimed at making money, it&#8217;s free software. The only thing that is actually labeled as a non-profit is the <a href="http://wordpressfoundation.org/">WordPress Foundation</a>. For those that don&#8217;t know what&#8217;s what in the world of WordPress, you should <a href="http://wpdaily.co/wpf-automattic/">read this article</a> by John Saddington on WPDaily.co which summarizes things nicely. Also, from what I&#8217;ve read, Ghost will have a hosting service similar to what WordPress.com provides. From the Ghost Kickstarter page.</p>
<blockquote><p>Users hosting with us = revenue = development = better software = more users = more revenue = more development = well&#8230; you get the idea.</p></blockquote>
<p>Sounds to me like there will be a commercial side to things. </p>
<p>I&#8217;m keeping a close eye on the progress of Ghost because not only are there a lot of people clamoring for a chance to use what&#8217;s been presented in the screenshots, but I&#8217;m interested in trying it out myself to see if it really does enhance the publishing process. However, comparisons between WordPress and Ghost should be reserved until an actual piece of software is released. I&#8217;m sure John is under a huge amount of pressure to deliver considering the amount of funding he will end up receiving. </p>

</ul>
<p>When I started using WordPress in June 2007</p>
<ul>

<p><strong><em>Q: First off, can you explain what ExperimentWith is?</em></strong></p>
<p>ExperimentWith.com was started as a testing site for our clients sites while we develop them. It had a nice ring to it and it allowed us to host testing on a separate but not local server. <p><strong><em>Q: What is going on behind the scenes of ExperimentWith.com and what software or hardware are you using to offer this service?</em></strong></p>
<p>Behind the scenes its a WordPress MU site running on one of our business class hosting accounts with access to ruby and sass for testing WordPress plugins and themes. We purchased five years of hosting from MidPhase and said what the hell, lets make it useful and if it becomes popular then we can keep it running.</p>
<p><strong><em>Q: Is ExperimentWith exclusively tailored for WordPress testing?</em></strong></p>
<p>We are tailored for WordPress testing but our server has elements running on it to allow for other WordPress integrations like Ruby, SASS, node.js, ect. We like experimenting with other projects out there to see how they can improve, on client specific projects.</p>
<p><strong><em>Q: It looks like the site is not yet open to the public, do you have a firm date in mind as to when that will happen?</em></strong></p>
<p>As for going public, we decided to initially start it up as an email us for about your project and if we like it and it won&#8217;t cause the end of the world or WordPress as we know it we will give you access. So not to dance around your question but Experiment With is a very dynamic project and we are open to help develop it to whatever the community needs.</p>

";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";s:5:"child";a:2:{s:0:"";a:5:{s:5:"title";a:1:{i:0;a:5:{s:4:"data";s:31:"WPTavern: MP6 Is Like Eye Candy";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"guid";a:1:{i:0;a:5:{s:4:"data";s:31:"http://www.wptavern.com/?p=6886";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"link";a:1:{i:0;a:5:{s:4:"data";s:83:"http://feedproxy.google.com/~r/WordpressTavern/~3/qPSAnt3ynuw/mp6-is-like-eye-candy";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:11:"description";a:1:{i:0;a:5:{s:4:"data";s:6644:"<p><a href="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/mp6logo.jpg" rel="thumbnail"><img class="alignright size-full wp-image-6888" alt="mp6 plugin header logo" src="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/mp6logo.jpg?resize=206%2C197" /></a>I&#8217;ve taken the <a href="http://wordpress.org/extend/plugins/mp6/">MP6 plugin</a> for a test drive and so far, my eyes love what they see. MP6 is an experimental plugin for testing out new ideas for the various UI elements that are within WordPress. My first reaction after activating the plugin was, <strong>WOW</strong>. The textual elements on the screen instantly became easier to read. The icons along with the colors of the left navigational menu were more pronounced and the post writing screen becomes easier to use. I couldn&#8217;t help but notice that I recognized some similarities between what was in MP6 versus the <a href="http://fortysevenrobots.com/wordpress/fluency-admin/">Fluency Administration theme</a>.</p>
<p>Here are a couple of comparison screenshots between MP6 and Fluency.</p>
<a href="http://www.wptavern.com/mp6-is-like-eye-candy#gallery-6886-1-slideshow">Click to view slideshow.</a>
<p>Personally, I like MP6 better than Fluency but I wanted to bring all of this up because back in 2007, <a href="http://wordpress.org/news/2008/12/coltrane/">WordPress 2.7 &#8220;Coltrane&#8221;</a> was released and its major feature was the user interface overhaul. In April of 2009, a <a href="http://wordpress.org/news/2009/04/poll-results/">poll was conducted</a> on WordPress.org where users voted for which design tweak they liked most. While the dark Fluency style contained the most votes, the conclusion of the poll was that overall, a darker color scheme was preferred. </p>
<div id="attachment_6927" class="wp-caption aligncenter"><a href="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/DesignPollResults.jpg" rel="thumbnail"><img src="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/DesignPollResults.jpg?resize=500%2C177" alt="2009 Poll Results" class="size-large wp-image-6927" /></a><p class="wp-caption-text">Poll Results From 2009</p></div>
<p>I reached out to Matt Thomas, one of the driving forces behind the MP6 plugin to ask him about these similarities between MP6 and Fluency as well as what the future might hold for MP6 items reaching the core of WordPress. </p>
<p><strong><em>Q: Regarding the style and design of MP6. Were there any inspirations taken from the Fluency admin theme? Back in 2009, there was a poll conducted on possible design Tweaks and the dark Fluency theme beat every other alternative in the poll <a href="http://wordpress.org/news/2009/04/poll-results/">http://wordpress.org/news/2009/04/poll-results/</a> and when I checked out MP6 today, it looked similar in ways, in others not so much.</p>
<p>Also, MP6 is an ongoing experiment but is its links tied back to that poll, knowing full well that there was definitely an audience that would love to see a darker back-end?</strong></em></p>
<p>There&#8217;s almost nothing we haven&#8217;t looked at in finding inspiration for MP6. Medium, Readmill, Twitter, Squarespace, Simple.com, we&#8217;ve got a vast collection of bits and pieces of UI that have inspired pieces of what you see in MP6. We didn&#8217;t directly rely on that survey or the comments from that development cycle, considering how much has changed since that time. But I&#8217;ve always been a fan of the Fluency plugin, though I never used it much since I blog on WordPress.com. It inspired my submissions to that 2.8 redesign discussion you mentioned, as well. And we were very glad to see that Dean is following the progress of MP6 and likes where it&#8217;s headed. <a href="http://make.wordpress.org/ui/2013/03/15/mp6-version-0-3/#comment-22786">http://make.wordpress.org/ui/2013/03/15/mp6-version-0-3/#comment-22786</a></p>
<p><strong><em>Q: While testing Fluency, it&#8217;s only compatible with WordPress 3.2. Keeping up with WordPress through the administration theme has to be tough work. Are there plans to keep this plugin updated along side WordPress for the foreseeable future or even perhaps add it to core as an option people could choose within their profile?</em></strong></p>
<p>Definitely, it&#8217;s difficult to keep our development coördinated with trunk just in the course of a single development cycle. Based on my preliminary conversations with Mark J and Nacin, I expect we&#8217;ll hand over MP6 to the core team around the beginning of the 3.7 development cycle to begin its implementation into core.</p>
<p><strong><em>Q: Last but not least did Dean J. Robinson contribute to anything in MP6?</em></strong></p>
<p>Didn&#8217;t contribute directly, but see above for the generous feedback he gave us.</p>
<p>Four years ago, users showed their support of a darker toned WordPress administration area. Is it no surprise then that <a href="http://macmanx.com/2013/04/19/the-future-wordpress-dashboard-on-wordpress-com/">so</a> <a href="http://www.doitwithwp.com/the-new-wordpress-dashboard-design-mp6-try-it-now/">many people</a> <a href="http://wordpress.org/support/view/plugin-reviews/mp6?filter=5">are loving the way</a> the back-end looks with MP6 installed? After going through major interface design changes in WordPress 2.3, 2.5, 2.7, and witnessing the frustration by fellow users, I commend the WordPress team on their new approach to design changes through a plugin versus outright putting it on our plates with a new release. In this way, we can avoid drastic interface changes from one release to the next and instead, use the plugin as a continual testing ground for enhancements. </p>
<p>To those who are using MP6, what are your thoughts? Would you like to see what&#8217;s presented within MP6 to be the next design iteration of the WordPress back-end or are you content with the current grey-blue color scheme?</p>

The gist of the post is an argument/discussion that&#8217;s been made over the years at different times. It boils down to figuring out a way to remove all the stuff that&#8217;s not needed in WordPress and coming up with a lean, modular core. Morten takes an interesting approach in that this new WordPress would consist of branches. You start with WP Core connected to different branches that contain specific functionality such as WP CMS or WP For Bloggers. In this way, those specific use case branches would have most of what you needed out of the box without having one WordPress package that contained everything for everyone. </p>
<p>On paper, I like the idea of a lean WordPress with modules that I enable or disable based on the functionality I need. Several years ago, I used a CMS called e107 that behaved in this manner. It was pretty cool at the time because if I wanted forums, I just enabled the forums module. If not, I left it disabled and the forums never loaded. Back in 2009, a lengthy discussion took place around the very topic of making WordPress more modular, <a href="http://lists.automattic.com/pipermail/wp-hackers/2009-February/024242.html">Less than Core, More than Plugins/Themes: Proposing Optional Modules</a>. Unlike creating different packages or branches of WordPress to fulfill specific needs, optional plugins could offer that functionality so that it wouldn&#8217;t be stuck in core.</p>
<blockquote><p>What I&#8217;d like to propose is the creation of a new concept for WordPress, the &#8220;Optional module.&#8221; Optional modules would be singular like the core and would be versioned with the core but they would not be included in the default installation. Optional modules would be &#8220;installed-on-demand&#8221; based on a plugin&#8217;s request and admin&#8217;s approval to install, and the version of the optional module installed would be determined by the version of the WordPress core that is installing it.</p>
<p>Having &#8220;Optional Modules&#8221; for WordPress such as I propose would allow progress in the direction of common infrastructure without bloating the core. And one of the first such optional modules could be a basic theming framework</p></blockquote>
<p>One of the biggest arguments against the idea is that the mostly volunteer based team that works on the core of WordPress is already pressed for time, (pardon the pun) and by creating these optional modules or plugins, it would significantly raise their work load to the point where it negatively affects their ability to work on WordPress core. That discussion took place around February of 2009. In <a href="http://wordpress.org/news/2009/12/canonical-plugins/">late 2009, early 2010</a> the term Canonical Plugin was created to differentiate these plugins from a normal WordPress plugin. A poll was conducted with different name choices with Core Plugins winning the top spot. At the time, a core plugin was defined as follows:</p>
<blockquote><p>Core plugins are community developed and encourage collaboration with multiple developers to satisfy the most popular functionality requests that would not make it into the Core of WordPress. These plugins would be developed alongside the core of WordPress to ensure compatibility, coding standards are met, secure code, etc. To highlight these plugins, a screen would be added to the plugins page in the back-end of WordPress to highlight these special plugins. Everything sounds great right? Let’s take a look at some of the possible unintended consequences this could have on the community.</p></blockquote>
<p>One of the only plugins that I know of that ended up with this Core Plugin status was called <a href="http://westi.wordpress.com/2009/12/26/giving-your-wordpress-a-check-up/">Health Check by Peter Westwood</a>. This plugin was used as a test bed for the idea of creating core plugins through collaboration. </p>
<p>I reached out to Aaron D. Campbell as he was one of the volunteers to generate effort and focus towards the Core Plugin project and asked him a few questions about the work that&#8217;s been completed since 2010 for Core Plugins.</p>
<p><strong>Jeff -</strong> On January 11th, 2010 you published on the Make.WordPress.core blog on creating a Core Plugin Infrastructure. The only core plugin that I know up which was an experiment was Health Check by Peter Westwood. Do you know of any others that have been created since 2010? Has there been much progress or work done towards the idea of Core Plugins since?</p>
<p><strong>Aaron -</strong>No. Post by E-Mail never really got off the ground. I&#8217;m hoping to get it kicked off at some point, possibly with a GSoC student this year (it was one of the ideas). Honestly, the best example of the CONCEPT of community driven development AROUND WordPress is probably _s. It&#8217;s a theme not a plugin, but it accomplished what we were envisioning</p>
<p><strong>Jeff -</strong> Well, Jetpack has sort of taken the Post by email feature and put it into a plugin. My guess would be that eventually that feature will be removed in WordPress like Links in favor of the Jetpack version.</p>
<p><strong>Aaron -</strong> I don&#8217;t think so. I don&#8217;t think any feature will ever be removed in favor of Jetpack.</p>
<p><strong>Jeff -</strong> I was under the impression that core plugins would enable WordPress to be able to take things that became not so widely used anymore and be able to put stuff like that into a plugin enabling a light nimble core. Therefor, decreasing the need to argue that WordPress is bloated and we need to fork it.</p>
<p><strong>Aaron -</strong> That&#8217;s certainly one of the goals of core plugins. However, Jetpack is not a core plugin.</p>
<p><strong>Jeff -</strong> So since 2010, not much work has been accomplished around the idea of Core Plugins but it&#8217;s still an idea that&#8217;s being kicked around and will possibly make headway in the future?</p>
<p><strong>Aaron -</strong> Yep. I think almost everyone agrees it&#8217;s a good idea, but having the hours needed to actually make it happen is a whole different thing.</p>
<p><strong>Jeff -</strong> Agreed. It was one of the issues immediately brought up in the hackers mailing list article first proposing the idea of these core modules. Time and effort which wouldn&#8217;t detract from the time and effort of Core itself </p>
<p>The issues that Morten has raised such as being stuck with old backwards compatibility code and future development of WordPress are justified but I don&#8217;t like the idea of different WordPress branches or cores. I don&#8217;t want to go through a process of figuring out which specific type of WordPress I need and I doubt new users will be able to easily grasp which one is best for them. We&#8217;ll end up needing flow charts, bar graphs and such to explain the differences. At the same time, I have noticed how much stuff WordPress has added over the years that I simply don&#8217;t use. Yet, the features I don&#8217;t use are the ones my neighbor does. It seems counter-productive to install plugins that remove unused functionality just so it&#8217;s completely out of my way. It also bothers me that so much of the unused portions of WordPress are loaded or even processed when they shouldn&#8217;t. This is why I&#8217;m for the idea of compartmentalizing WordPress to the point of simply being able to turn things on or off. Then again, I can see how having a module for the simple task of disabling or enabling image editing in WordPress could be cumbersome. </p>
<p>I want to see WordPress continue on for another 10 years but at the same time, I don&#8217;t want it to bring everything from the previous 10 years with it. Although I haven&#8217;t mentioned the word bloat, this post from 2009 is relevant in that it discusses the question, <a href="http://www.wptavern.com/what-is-wordpress-bloat">What is WordPress bloat</a>? </p>
<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ghostlogo.jpg" rel="thumbnail"><img src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/ghostlogo.jpg?resize=313%2C135" alt="ghostlogo" class="aligncenter size-full wp-image-6854" /></a></p>
<p>I&#8217;d be re missed if I didn&#8217;t mention <a href="http://tryghost.org/">Ghost</a> somewhere in this conversation. <a href="http://john.onolan.org/ghost/">Ghost is a vision</a> proposed by John O&#8217;Nolan that aims to get back to the roots of simple, easy, publishing. I&#8217;m excited about Ghost and joined in with the crowd of wanting to see it in action. At this point, Ghost is just a bunch of pretty screenshots but I like what I see. John started a <a href="http://www.kickstarter.com/projects/johnonolan/ghost-just-a-blogging-platform">Kickstarter campaign</a> to acquire 25,000 Great British Pounds to get the project off the ground. Not only did he reach that goal within hours of starting the campaign, he has since quadrupled the goal amount with 19 days left to go. What this tells me is that there is pent-up interest in seeing a platform that gets back to the basics of publishing. Many of the names on the backers list are long-time WordPress users. </p>
<p>There are techniques and methods to get WordPress back to its roots of being the easiest software to use to publish words to the web. Forking is not one of those methods. Are you willing to stick with WordPress through the long haul or are you ready to pack your bags and jump ship the minute something easier and better looking is released?</p>

";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";s:5:"child";a:2:{s:0:"";a:5:{s:5:"title";a:1:{i:0;a:5:{s:4:"data";s:49:"WPTavern: Searching For A Better WordPress Search";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"guid";a:1:{i:0;a:5:{s:4:"data";s:31:"http://www.wptavern.com/?p=6819";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"link";a:1:{i:0;a:5:{s:4:"data";s:101:"http://feedproxy.google.com/~r/WordpressTavern/~3/19hutSdYz4I/searching-for-a-better-wordpress-search";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:11:"description";a:1:{i:0;a:5:{s:4:"data";s:5260:"<p>On May 6th, the WordPress VIP team blog <a href="https://vip.wordpress.com/2013/05/06/vip-search/">announced</a> a new VIP service they&#8217;ll be offering centered around search. While at first it may be surprising that the default search in WordPress was not used in their latest project, the fact is, WordPress search in its current state can not perform complex search queries without using a considerable amount of computational resources which spells doom for users on shared webhosting. The VIP Search Add-on uses third party software called <a href="http://www.elasticsearch.org/">Elasticsearch</a> which is open source and can be used by anyone. However, upon looking at the requirements and the methods of setting it up, it&#8217;s only meant for those that have the ability to run software on their hosting accounts either through a dedicated box or VPS. By using Elastic search, the VIP team was able to provide search capabilities such as being able to filter between the different types of content, categories, tags, authors, date ranges and more. To see the advanced search capabilities in action, check out <a href="http://kff.org/search/">http://kff.org/search/</a>. I reached out to Paul Maiorana, Vice President of Platform Services for Automattic but was unable to get a response I could share with the public. </p>
<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/VIPsearch.jpg" rel="thumbnail"><img src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/VIPsearch.jpg?resize=300%2C136" alt="VIP Search box" class="aligncenter size-medium wp-image-6820" /></a></p>
<p>I briefly conversed with other members of the WordPress community on Twitter on why WordPress does not already have those advanced search queries out of the box. The gist of the conversation was that performing such intensive searches would be too much for the limited resources shared hosting provides. It was also mentioned that most of the functionality can be achieved using Google Custom Search. In fact, Matt Mullenwegs personal WordPress site Ma.tt runs Google Custom Search. </p>
<p>Interestingly enough, back in 2009, Matt <a href="http://ma.tt/2009/07/acquia-searc/">published a post</a> that highlighted a new hosted service called Acquia Search that Drupal users could tap into. At the time, commenters urged Matt to continue to improve the search capabilities in WordPress as some users were not fans of relying on a third party service. Matt&#8217;s response was:</p>
<blockquote><p>Yes and there is a Google Summer of Code project on this right now, but as with video processing we’re limited by the resources available to us by default and it’s something that could be fairly efficiently done as a remote service.</p></blockquote>
<p>If you&#8217;re interested, you can <a href="http://wpsoc09.wordpress.com/2009/03/29/extended-wordpress-search-engine/">see the proposal</a> for that Google Summer Of Code project here. <a href="http://andy.wordpress.com/2009/08/21/wordpress-search-plugin/">According to Justin&#8217;s mentor Andy Skelton</a>, Justin Shreve was successful in completing his search API project which the results can be found within the <a href="http://wordpress.org/extend/plugins/search/">plugin repository</a> except that the plugin has not been updated for over two years. After asking around to see if the search API ever made it to core, it appears not. </p>
<p>So while I don&#8217;t see any major improvement taking place in the near future for the core implementation of search, there are a few plugins out there that improve the situation. In fact, if you conduct a search of the WordPress plugin repository using the <a href="http://wordpress.org/extend/plugins/search.php?q=search">keyword of search</a>, 16 results show up although not all of the results are specifically tailored for the search functionality of WordPress. I personally have not yet taken the time to evaluate the different plugins but I have heard from a number of users that <a href="http://wordpress.org/extend/plugins/relevanssi/">Relevanssi</a> is a good solution. Of course, there is always <a href="http://wordpress.org/extend/plugins/google-custom-search/">Google Custom Search</a> which has a proven track record.</p>
<p>Do you think having better search functionality in WordPress is worth paying for? Have you installed any plugins or have manually improved the search capabilities on your website? Let me know your thoughts in the comments on the state of search in WordPress. </p>

";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";s:5:"child";a:2:{s:0:"";a:5:{s:5:"title";a:1:{i:0;a:5:{s:4:"data";s:50:"WPTavern: Rolling Your Own WordPress With WPRoller";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"guid";a:1:{i:0;a:5:{s:4:"data";s:31:"http://www.wptavern.com/?p=6806";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"link";a:1:{i:0;a:5:{s:4:"data";s:102:"http://feedproxy.google.com/~r/WordpressTavern/~3/fAU7A6fErwI/rolling-your-own-wordpress-with-wproller";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:11:"description";a:1:{i:0;a:5:{s:4:"data";s:5712:"<p><a href="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wprollerlogo.jpg" rel="thumbnail"><img src="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wprollerlogo.jpg?resize=261%2C64" alt="wp roller logo" class="alignright size-full wp-image-6807" /></a>Almost four years ago, <a href="http://lists.automattic.com/pipermail/wp-hackers/2009-September/027659.html">the idea</a> of having <a href="http://www.wptavern.com/custom-install-profiles-sounds-like-a-cool-idea">custom WordPress installation profiles</a> was brought up. The initial idea was to change core around so that developers could pre-configure WordPress with a set of activated plugins while also being able to change/activate the default theme. The idea was championed at the time but the implementation was up for debate. Fast forward to today and there is now a free service called <a href="http://wproller.com/">WPRoller</a> which looks like a great solution to the problem, without having to do anything to the core of WordPress. </p>
<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-wpversion.jpg" rel="thumbnail"><img src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-wpversion.jpg?resize=300%2C138" alt="WPRoller Choosing Which Version Of WordPress To Install" class="aligncenter size-medium wp-image-6808" /></a></p>
<p>Without needing to register an account, you simply scroll down the page and select which version of WordPress you&#8217;d like to use, either the latest stable version or the nightly build. Then, scroll through the various categories of plugins and select the ones you&#8217;d like to install along with WordPress. While at first I was disappointed that I couldn&#8217;t search the repository and just add whatever plugin I wanted, it then dawned on me how much more sense it makes to just list some of the most popular plugins in their respective categories. </p>
<p><a href="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-choosingplugins.jpg" rel="thumbnail"><img src="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-choosingplugins.jpg?resize=300%2C171" alt="Choosing WordPress Plugins" class="aligncenter size-medium wp-image-6809" /></a></p>
<p>During the initial launch of the service, users were unable to add plugins that were not already on the list. However, the site has recently undergone some updates so that you can now create a profile. Users can now upload themes and plugins to their WPRoller profile that will also be rolled into their custom WordPress installation. A killer feature if you ask me. Any theme or plugin that is uploaded to your profile is private. No one else can access that information which should generate a sigh of relief from commercial theme and plugin companies. </p>
<p><a href="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-profile.jpg" rel="thumbnail"><img src="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/wproller-profile.jpg?resize=300%2C138" alt="WPRoller Profiles" class="aligncenter size-medium wp-image-6810" /></a></p>
<p>Once you&#8217;ve gone through and created your WordPress package, you&#8217;ll have the opportunity to save it for future re-use. Being able to have multiple WordPress packages saved to your profile provides the ability to easily have different installations based on a clients/users needs. All of this is accomplished without modifying the core of WordPress which again, is a nice feature in and of itself. Depending upon how many plugins are added to a WordPress package, the zip file will range between 5-100 megs in size. So if you end up timing the 5 minute WordPress install claim, don&#8217;t add the time it takes to upload the zip file as that&#8217;s not fair.</p>
<p>I can&#8217;t believe this website is free. As far as I can tell, there doesn&#8217;t appear to be a limit to the amount of plugins in the package, size of the zip file, or the number of packages you can save to your profile. Oh, if you&#8217;re wondering whether or not the plugins packaged with your WordPress install are up to date, <a href="http://wproller.com/blog/wproller-com-updates/plugins-updated-3-27-2013/">WPRoller says</a> they have a nightly Cron job running to make sure the latest versions end up in your installation zip file. While their last blog post was on April 10th, WPRoller has plans to allow custom install.php, wp-config.php and .htaccess files as well as installation starter packages for particular use case segments.</p>
<p>So what I want to know from you is whether or not you&#8217;ve used this service and if so, has it really made your life of launching WordPress based sites easier? After all, this was only a pipe dream 4 years ago, and while plugins have been released since then to sort of create various WordPress install profiles, I just love the simplicity of how WPRoller executes its mission.</p>

The event took place on the Stark State College Campus within the Center for Entrepreneurship and was coordinated by Joe Rozsa. The venue offered plenty of seating, multiple classrooms for sessions and outlets galore for charging electronic devices. Although the WiFi at the event was surprisingly low speed considering the venue, it was&#8217;nt enough to notice when tweeting or pushing out images. In fact, there was little in the way of problems with the WiFi. </p>

<a href="http://www.wptavern.com/first-wordcamp-north-canton-a-success/attachment/114" title="114"><img src="http://i0.wp.com/www.wptavern.com/wp-content/uploads/2013/05/114.jpg?resize=150%2C150" class="attachment-thumbnail" alt="114" /></a>

<p>One of the cool aspects of this event is that it was the first WordCamp to have a Zero waste strategy. Everything used during the event was either recyclable or fully bio-degradable. There was a station set up in the back of the main room staffed by volunteers to help make sure things were put in the proper place. Having zero waste at a WordCamp event is beneficial no matter how you look at it. From what I heard, Joe Rozsa will be putting together a tutorial for how he accomplished this feat which will be shared on WordCamp.org for other organizers to take advantage of. </p>
<p>One of the highlights for me was the lunch prepared by <a href="http://ermannospizza.com/">Ermannos pizza</a>. They brought in a selection of pizzas but my favorite was their Margherita which contained Extra Virgin Olive oil Infused with Garlic, their Triple Blend Cheese, Roma Tomatoes, Fresh Basil. Topped with Parmesan &#038; Romano Cheese, Italian Herbs &#038; Seasonings, and Balsamic Glaze. This pizza was awesome and when I&#8217;m back in the North Canton area, I&#8217;ll definitely be stopping by for some more. </p>
<p><a href="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/119.jpg" rel="thumbnail"><img src="http://i2.wp.com/www.wptavern.com/wp-content/uploads/2013/05/119.jpg?resize=224%2C300" alt="119" class="aligncenter size-medium wp-image-6776" /></a></p>
<p>One other treat that I&#8217;ve not seen at any WordCamps is S&#8217;mores. Ermanno&#8217;s also baked ravioli stuffed s&#8217;mores that were fluffy, gooey pastries filled with marshmallow and chocolate. While I managed to eat two of them and then take a few home, I know of a few other attendees that couldn&#8217;t stop eating them. </p>
<p><a href="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/120.jpg" rel="thumbnail"><img src="http://i1.wp.com/www.wptavern.com/wp-content/uploads/2013/05/120.jpg?resize=223%2C300" alt="120" class="aligncenter size-medium wp-image-6799" /></a></p>
<p>Overall, the event was a major success for it being the first of it&#8217;s kind for the North Canton area. The sessions I attended were presented by people passionate about the subject they were talking about. Most of the people in the audience were either new to WordPress or had little to no development experience. I also wanted to point out that a lot of the attendees at this event were women. If s&#8217;mores didn&#8217;t cap off the camp in WordCamp, tents and sleeping bags were among some of the prizes given away at the end of the day. Joe Rozsa and his team of volunteer staff members are to be commended for a job well done.</p>

<h2>Hotkeys</h2>
<p>In the <code>feature/hotkeys</code> branch<sup id="fnref:1"><a href="http://alexking.org/blog/topic/wordpress/feed#fn:1" rel="footnote">1</a></sup> in the <code>ui</code> submodule, you&#8217;ll find some additional keyboard shortcuts:</p>
<ul>
<li>shift+h = navigate to home</li>
<li>shift+n = new entry</li>
<li>shift+f = set focus to search field</li>
</ul>
<p>Make sure to update submodules as the <a href="https://github.com/jeresig/jquery.hotkeys">jQuery Hotkeys library</a> has been added.</p>
<h2>Capsule Server Queue</h2>
<p>In the <code>feature/queue</code> branch you&#8217;ll find an implementation of a very basic &#8220;send at least once&#8221; queue so that pushing posts to Capsule Server is no longer a blocking action (it makes saves faster). Another benefit to this feature is better support for using Capsule while offline. Saved entries are stored in the queue and sent to Capsule Server once Capsule is online again. To test this out, you&#8217;ll need to update your Capsule Server to the latest code in the <code>develop</code> branch.</p>
<p>We&#8217;ve also <a href="https://github.com/crowdfavorite/wp-capsule/issues?page=1&state=closed">fixed a few bugs and incorporated a few submitted enhancements</a>. Thanks to everyone who has contributed.</p>

It has been an amazing 10 years, during which WordPress evolved from a simple blogware to a very full-featured CMS (Content Management System), used to power some of the biggest and most popular web sites on the internet. All over the world, people are <a title="Party on, Wayne!" href="http://wordpress.org/news/2013/04/save-the-date-may-27/">planning celebrations</a>. As much as I like a good party, I thought this would also be a good time to celebrate WordPress by actually using WordPress &#8212; for blogging.</p>
<p>So much of our content sharing these days tends to be via centralized &#8220;social&#8221; platforms like Twitter, Facebook, Google+, Tumblr, etc., that some of us neglect our blogs. I know <em>I&#8217;m</em> guilty of that. But this anniversary is a good excuse for the WordPress community to take a few minutes to consider their time with our favorite website building tool. Here&#8217;s what I propose&#8230;</p>
<p><span id="more-64213"></span><br />
Think about the past 10 years, and your time with WordPress. Maybe, like myself, you&#8217;ve been using WordPress since the beginning, or perhaps you just tried it out for the first time last month. But consider the events that stand out in your memory during that time. Now, write a blog post about it. Wherever you fall on the timeline, you should probably be able to write something about one or more of the following openings:</p>
<ul>
<li>&#8220;When WordPress was first released in 2003..&#8221;</li>
<li>&#8220;When I first started using WordPress&#8230;&#8221;</li>
<li>&#8220;Since I first started using WordPress&#8230;&#8221;</li>
</ul>
<p>So, for example, I&#8217;ve been using WordPress pretty much since the beginning. I didn&#8217;t actually switch my blog until September 2003, but I&#8217;d been committing to core before that, so I might list things like,</p>
<p>When WordPress was released in May 2003:</p>
<ul>
<li>I was 36 years old</li>
<li>I had been married for less than 2 years</li>
<li>I already had nearly 10 years of web development experience</li>
<li>My Pentium 4 laptop with 1GB of RAM was a screaming machine</li>
</ul>
<p>When I first started using WordPress in September 2003&#8230;</p>
<ul>
<li>The Ellen Degeneres show had just debuted on TV</li>
<li>The US Treasury redesigned the $20 bill</li>
<li>George W. Bush was in his first term as President</li>
<li>SCO and IBM were locked in a legal battle over ownership of Linux source code</li>
<li>My day-job was doing web development in VBScript and PerlScript on IIS4 servers</li>
<li>Many people didn&#8217;t have cell phones, and the iPhone was still 4 years away</li>
<li>Mozilla replaced Netscape</li>
<li>The term &#8216;AJAX&#8217; (Asynchronous JavaScript and XML) didn&#8217;t exist yet</li>
<li>The Ubuntu distribution of Linux didn&#8217;t exist yet</li>
<li>PHP5 was brand spanking new</li>
<li>Facebook was only a few months old, and I hadn&#8217;t even heard of it yet</li>
<li>Geocities was still a big thing</li>
</ul>
<p>Since I started using WordPress&#8230;</p>
<ul>
<li>There have been 2 Summer Olympics and 2 Winter Olympics</li>
<li>I moved from southeast Alabama to Atlanta, Georgia</li>
<li>I became a homeowner</li>
<li>I have changed jobs at least 8 times, if you include short-term contracts</li>
<li>I have had another daughter, who is now in 2nd grade</li>
<li>Apple switched from Motorola PowerPC to Intel processors</li>
<li>My family adopted a dog and two cats</li>
<li>There have been 5 iterations of the iPhone and 3 of the iPad</li>
<li>Myspace went from top-of-the-heap to also-ran to come-back-kid</li>
</ul>
<p>There could be all sorts of life, historical, or technological events listed, but anyhow, you get the idea.</p>
<p>It&#8217;s nothing complicated, but I just thought this would be a really fun thing to give a lot of people something common to write about, and actually use our WordPress sites for publishing content. I suggest we use the tag <strong><code>wp10</code></strong> for all the posts (hashtag <strong><code>#wp10</code></strong> for Twitter and such). Also, in addition to writing your own post, make sure to seek out posts by others, and link to your favorites!</p>

";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";s:5:"child";a:2:{s:0:"";a:5:{s:5:"title";a:1:{i:0;a:5:{s:4:"data";s:47:"Donncha: mfunc in WP Super Cache 1.4 and beyond";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"guid";a:1:{i:0;a:5:{s:4:"data";s:29:"http://ocaoimh.ie/?p=89498328";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:4:"link";a:1:{i:0;a:5:{s:4:"data";s:68:"http://ocaoimh.ie/2013/05/01/mfunc-in-wp-super-cache-1-4-and-beyond/";s:7:"attribs";a:0:{}s:8:"xml_base";s:0:"";s:17:"xml_base_explicit";b:0;s:8:"xml_lang";s:0:"";}}s:11:"description";a:1:{i:0;a:5:{s:4:"data";s:5670:"<p>WP Super Cache is a full page caching plugin for WordPress. It creates static pages that are served quickly by the web server. Sometimes however, users still want parts of their pages to remain dynamic and be non static. That&#8217;s where <a href="http://wordpress.org/extend/plugins/wp-super-cache/faq/">mfunc, mclude and dynamic-cached-content</a> came in.</p>
<p><strong>Security</strong></p>
<p>Unfortunately it was reported recently that remote visitors to sites using the plugin could execute any code they like by simply leaving a comment containing the right mfunc code. These functions are now disabled by default, and a filter removes harmful code from comments but if enabled they pose a security risk. I considered adding a security code to the mfunc tag but unfortunately the best way of dealing with this problem is to replace it completely with something different. The next release of the plugin will do away with mfunc, mclude and dynamic-cached-content entirely. </p>
<p><strong>The new dynamic cache system</strong></p>
<p>The <a href="http://ocaoimh.ie/the-development-version-of-wp-super-cache/">development version of WP Super Cache</a> has already been updated with a new filter based system. It uses a cacheaction filter called &#8220;wpsc_cachedata&#8221;. This filter runs when a page is first cached and also when a cached page is subsequently served. It also runs when caching is disabled for known users, something that has always been broken when using mfunc.</p>
<p>Almost all the data that is displayed on your website will run through the cacheaction filter &#8220;wpsc_cachedata&#8221;. When a page is first cached, the data that is shown to the first visitor of that page goes through that filter. The second visitor gets a cached page and that page too goes through the filter. What this allows us to do is define a template tag (or more than one) that a function hooked on that filter can search for in the filtered data. It can replace that tag with some other text, usually derived from code that has to run on each request. The visitor is then shown the page with the replaced tag.</p>
<p><strong>Example code</strong></p>
<p>The readme.txt hasn&#8217;t been updated yet but an example plugin, <a href="http://svn.wp-plugins.org/wp-super-cache/trunk/plugins/dynamic-cache-test.php">dynamic-cache-test.php</a> is included in WP Super Cache. It&#8217;s fairly simple but it&#8217;s documented so it should be easy enough to follow. A template tag is inserted at the bottom of the page using the wp_footer action, and a filter then replaces that tag with text and the current server time. That test plugin replaces mfunc code that would look like this, excluding the necessary code to hook on to wp_footer and print it.</p>
<p><code>&lt;!--mfunc echo "&lt;!-- Hello world at " . date( 'H:i:s' ) . " -->"; --><br />
&lt;?php echo "&lt;!-- Hello world at " . date( 'H:i:s' ) . " -->" ?><br />
&lt;!--/mfunc--></code></p>
<p>WP Super Cache has it&#8217;s own action hooks using add_cacheaction() and do_cacheaction(), and work like WordPress actions or filters. The reason the plugin needs those is because they are available <em>before</em> WordPress is loaded. They allow developers to hook into the plugin from the very start of the PHP process and modify how it works using plugins. Those plugins are usually copied into wp-super-cache/plugins/ but I encourage you to move that directory elsewhere because when WordPress updates the plugin it will delete any custom changes you make. The next time a new version of WP Super Cache comes out WordPress will delete the wp-super-cache folder, replacing it with the new update. In your wp-config.php set <code>$wp_cache_plugins_dir</code> to the location of the new plugins directory.</p>
<p>If you use this filter system in your own plugin for distribution do not <em>ever</em> define the template tag for the user. Let the user decide what it is or generate a random tag and save it somewhere. It&#8217;s important to keep the tag secret so visitors cannot trigger your function maliciously. It is however better than the remote user running any code they like as was the case with mfunc!</p>
<p>I hope to release a new version with this code late next week. If your plugin or site uses mfunc please download the <a href="http://ocaoimh.ie/the-development-version-of-wp-super-cache/">development version</a> on a test server and start the process of updating your code.<br />
On the other hand, if you don&#8217;t want to update your mfunc tags you could try <a href="http://wordpress.org/extend/plugins/w3-total-cache/">W3 Total Cache</a> instead. It uses the mfunc tag with a secret code.</p>
